개인정보 처리방침

최종 수정일: 2026-05-11

sticky(이하 "본 앱")는 사용자의 개인정보를 최소한으로만 다루는 것을 원칙으로 합니다. 본 방침은 본 앱이 어떤 데이터를 어떻게 수집·저장·전송하는지를 설명합니다.

1. 본 앱이 다루는 데이터

  • 일정·할 일·루틴 콘텐츠 — 사용자가 직접 입력한 제목·메모·날짜·시간·우선순위·반복 정보.
  • 외부 캘린더 데이터(선택) — 사용자가 권한을 허용한 경우에 한해 Apple 캘린더 / Google 캘린더의 일정을 읽어 표시합니다.
  • 알림 설정 — 사용자가 지정한 시간에 로컬 알림을 띄우기 위한 정보.

본 앱은 위치 정보, 연락처, 사진, 마이크, 광고 식별자(IDFA)를 수집하지 않습니다.

2. 데이터의 저장 위치

사용자가 입력한 모든 콘텐츠는 사용자의 iCloud 계정에 저장됩니다(Apple의 CloudKit). 이 데이터는 사용자의 Apple ID로만 접근할 수 있으며, sticky 개발자가 사용자의 데이터를 들여다볼 수 있는 경로는 존재하지 않습니다.

네트워크 사용이 불가하거나 iCloud가 비활성화된 경우에는 기기 내 로컬 저장소에만 저장됩니다.

3. 외부로 전송되는 데이터

본 앱은 다음 외부 엔드포인트와 통신합니다. 각각의 목적과 전송 범위는 다음과 같습니다.

  • Apple iCloud (CloudKit) — 사용자의 일정·할 일·루틴 동기화. Apple의 표준 서비스이며 sticky 개발자가 접근할 수 없습니다.
  • Google OAuth / Google Calendar API(선택) — 사용자가 Google 캘린더 연동을 활성화한 경우에만, 사용자의 Google 계정에서 캘린더 일정을 읽기 전용으로 가져옵니다. 본 앱은 가져온 일정을 외부에 다시 전송하지 않습니다.
  • 한국 공공데이터포털 (data.go.kr) — 한국 공휴일 정보를 가져옵니다. 연도 정보 외에 사용자 식별 정보는 전송하지 않습니다.
  • jsDelivr CDN (cdn.jsdelivr.net) — 강제 업데이트 안내를 위한 정적 JSON 파일을 가져옵니다. 사용자 데이터는 전송하지 않습니다(일반 GET 요청).

본 앱은 광고 네트워크나 분석 SDK(예: Google Analytics, Firebase Analytics 등)를 사용하지 않습니다.

4. 권한 요청

  • 알림 — 일정 시작 전과 루틴 시간에 알려드리기 위함.
  • 캘린더(EventKit) — Apple 캘린더 일정을 함께 표시하기 위함(선택).

각 권한은 거부하거나 iOS 설정에서 언제든 철회할 수 있으며, 거부 시에도 본 앱의 핵심 기능은 정상 동작합니다.

5. 보유 기간 및 삭제

사용자가 직접 일정·할 일·루틴을 삭제하면 즉시 제거되며, 동기화된 다른 기기에서도 함께 사라집니다. 앱을 삭제하면 기기 내 로컬 데이터는 함께 삭제됩니다. iCloud에 저장된 데이터를 완전히 지우려면 iOS 설정 → Apple ID → iCloud → sticky의 "데이터 삭제"를 사용해 주세요.

6. 13세 미만 사용자

본 앱은 특정 연령대를 대상으로 설계되지 않았으며, 13세 미만 사용자의 개인정보를 의도적으로 수집하지 않습니다.

7. 방침 변경

본 방침은 앱 기능 변경 시 갱신될 수 있습니다. 변경 시 본 페이지 상단의 "최종 수정일"을 업데이트합니다. 중대한 변경이 있을 경우 앱 내 공지로 안내합니다.

8. 문의

개인정보 처리에 대한 문의는 info@dolgo.org로 보내주세요.
